Limitations

The following are known limitations and bugs that are not sufficiently mission-critical to have been addressed.

  1. Applet limitations The GR applets do not work well under Java 1.5.0-beta. Use 1.4.2-04 instead.
  2. JNI User interface problems The JNI User interface has problems creating new JNI models. If you follow the instructions in JNIUtilities, then eventually you will get a NameDuplicationException.

    The resulting model will be missing 

    <property name="output" class="jni.Argument" value="false,false,true,double">
</property>
    We are working with the original authors on a solution.
    In the short term, one workaround is to edit the xml file directly and add the appropriate property by hand.

    Another issue is the the JNI interface will not work wit Hierarchical classes, subclasses, and inner classes.

  3. Matlab interface has problems work with gcc-3.3 Under Windows, the Matlab interface has problems at runtime with gcc-3.3.
    Under Windows, we recommend using gcc-3.2.
  4. Copernicus does not work will if there are spaces in $PTII If Ptolemy is installed in a directory that has a space in the pathname, then the Copernicus code generator does not work well.
    The problem is because Soot uses the StringTokenizer to process options. The space character is a delimiter, which causes problems.
  5. Plot windows don't resize when control panel is resized. If a model with plotters is executed using the run button on the toolbar, and then re-executed using the run control panel accessed through the View menu, then resizing the run control panel does not cause the plots to resize to fit. Workaround: Close the model and re-open it, and then run first using the run control panel.
  6. Control click and Mac OS X.

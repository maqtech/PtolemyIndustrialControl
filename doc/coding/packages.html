Ptolemy II Packages

Places to go
  • Directory Structure
    • Ptolemy II uses the Java package system to group classes.
  • Bidirectional dependencies should be avoided. If two packages are dependent upon each other, it makes it hard to determine which package to compile first. Below are the possibilities
  • Remove the dependency
  • Combine the two packages into one package
  • Move the dependency into a third package, which will probably be a subpackage of one of the packages.
  • A subpackage is a package that is beneath a parent package in the directory hierarchy. For example pt.kernel.util is a subpackage of pt.kernel. The makefiles are structured so that we run make in the subdirectories first so in general, it is best if parent packages depend on their subpackages.

Discrete Event Interruptible Processor Simulation

This applet uses the Ptolemy II DE domain architecture to simulate the behaviour of an interruptible processor. Without interrupts, each demand takes a fixed minimum service time to serve. With interrupts, the service time is increased by the number of interrupt events during the service period multiplied by a fixed interrupt service time.

The plot shows the sampled processor output. The input to the processor is a sequence of DoubleToken starting at 0.0 at time 0.0 and incremented by 1.0 every unit time. The output are sampled to model a latched output. If there is no loss of events, then the plot is a straight line. Otherwise, there will be flat regions.

Different parameters for the processor model can be chosen. MST (Minimum Service Time) indicate the amount of time needed to process an event, if there is no interrupt. IST (Interrupt Service Time) is the amount of time needed to process an interrupt. Note that the interrupts preempt the inputs. Lambda represents the mean inter-arrival time of the Poisson process modeling the interrupt arrival. Lastly, stop time is just the time when simulation stops.

This applet is implemented by InterruptApplet.java.

The simulation consists of a Clock Actor, a Ramp Actor, DEProcessor Actor, DEInterruptibleServer Actor, DESampler Actor, and a DEPoisson Actor connected to a Plot Actor.

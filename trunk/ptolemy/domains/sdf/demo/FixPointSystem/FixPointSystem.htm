Fixed-point conversion and overflow demo

The follow plot shows a the numbers generated by a Ramp actor. The ramp generates new double tokens with a .1 increment. These tokens are being converted into a Fixed-point token with a given precision.

The precision is equal to (2.1) which means that 2 bits are used to represent the integer bits and 1 bit is used to represent the fraction bit. So the set of numbers that can be represented with this precision is 

{ -2.0, -1.5, -1.0, -0.5, 0.0, 0.5, 1.0, 1.5 }
As a consequensem the diagram shows the quantization errors causes by the fixed precision.

In the quantization, three different modes can be selected:

  1. Saturate
  2. Rounding
  3. Truncate

The assumptions are:

  • Additive Gaussian noise with standard deviation 0.1.
  • 100 % excess bandwidth square-root raised-cosine pulse.
  • Matched filter in the receiver.

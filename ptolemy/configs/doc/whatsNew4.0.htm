Some capabilities in Version 4.0 require additional software and/or hardware

  • Using the serial port (see SerialPort actor documentation).
  • X10 power-line network (see X10Interface actor documentation).
  • The Matlab Expression actor how takes a String Parameter instead of a StringAttribute This means that you can now reference parameters in scope using the $name or ${name} syntax. For instance, if there is a parameter named foo, you can specifcy a Matlab expression like: 
        output = fft(in, $foo)
    Before, you would have to have read foo in a Const actor and provided it as input to the Matlab expression actor.

Major Features that are new in Ptolemy II 4.0

Below are new features that were not present in Ptolemy II 3.x
  • Classes and inheritance at the block diagram level (Edward A. Lee)
  • Lifecycle management components (Yang Zhao)
  • Unit system rework (Rowland Johnson)
  • Copernicus/Actor (Steve Neuendorffer)
  • Updates to Vergil, the user interface for building models (Steve Neuendorffer, Edward A. Lee)
  • Decorative elements in block diagrams (boxes, etc.)
  • Much smaller XML files
  • Icon editor (and animated icons)
  • Expert mode parameter editing
  • vergil -help lists available configurations.
  • Full-screen mode for use in presentations.
  • Single Window Mode (J. Blanc (Thales), B. Masson (Thales))
    If vergil -single is invoked, then a single window interface is used. This interface is experimental.
    Limitations:
    • The -single argument should be the first argument.
    • The plot windows come up outside of the single window interface.
  • XMLToken (Yang Zhao)
  • Sandbox support
    (Christopher Hylands Brooks)
    Vergil has better support for operating in a security sandbox. For details, see the Ptolemy II Sandbox documentation.
